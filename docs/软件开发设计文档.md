## 1. 引言

### 1.1 背景

本项目在两个既有工具的基础上融合、重构而成：

- **resting-screen（休息屏助手）**：一个常驻托盘的后台程序，通过全局快捷键 `Alt + L` 触发"息屏"或"系统锁屏"。
- **black_screen / BlackLock（黑屏锁）**：一个纯 Win32 C 编写的极简程序，双击运行后用一块覆盖全屏的黑色窗口把桌面遮住，输入密码回车解锁。

本项目取两者之长：既要 resting-screen 的**全局快捷键 + 开机自启 + 常驻后台**能力，又要 black_screen 的**"真黑屏遮挡"而非息屏**的锁定方式和**独立密码解锁**逻辑，同时进一步追求**无界面、配置文件驱动、单文件、极小体积**。

### 1.2 术语表

| 术语 | 含义 |
|------|------|
| 息屏 / Monitor Off | 让显示器进入省电待机（`SC_MONITORPOWER`），动一下鼠标即唤醒，不阻止系统睡眠 |
| 黑屏锁 / Black Lock | 用一块黑色置顶窗口覆盖整个虚拟桌面，动鼠标不解锁，需输入密码；显示器仍在工作 |
| 系统锁屏 / LockWorkStation | 调用 Windows 自带锁屏（Win+L 效果），走系统账户密码 |
| 常驻进程 / Resident Daemon | 无窗口、无托盘、在后台监听快捷键的进程 |
| 虚拟桌面 / Virtual Screen | 所有显示器拼接成的整体坐标空间（多屏场景） |
| Message-only 窗口 | 仅用于收发消息、不可见、不参与绘制的窗口（`HWND_MESSAGE`） |

### 1.3 参考资料

- resting-screen 源码：`https://github.com/Hunlongyu/resting-screen`
- black_screen 源码：本地 `black_screen-main`（`src/lock.c`，398 行）
- Win32 API 文档：`SetWindowsHookEx`、`RegisterHotKey`、`SetThreadExecutionState`、`BCrypt`、`GetPrivateProfileString`

## 2. 参考项目分析

本节整理两个参考项目各自实现了哪些主要功能，作为新工具的能力来源与取舍依据。

### 2.1 resting-screen（休息屏助手）

**技术栈**：C++、CMake、Inno Setup 打包，约 78 KB，内存 1.8–6.5 MB，支持 x86/x64。

**已实现主要功能：**

| 功能 | 实现方式 |
|------|----------|
| 全局快捷键 `Alt + L` | `RegisterHotKey()` 注册，主窗口过程 `WndProc` 收 `WM_HOTKEY` 后调用 `RestingScreenFn()` |
| 托盘图标 + 右键菜单 | `Shell_NotifyIcon(NIM_ADD)`；菜单含"锁屏 / 快捷键开关 / 开机自启开关 / 退出" |
| 息屏模式 | 广播 `WM_SYSCOMMAND` + `SC_MONITORPOWER`（参数 2）关闭显示器 |
| 系统锁屏模式 | 息屏前调用 `LockWorkStation()` |
| 配置文件 | `%APPDATA%\screen\config.ini`，`[Application]` 段：`version / shortcut / lock / autostart` |
| 开机自启 | 读写注册表 `HKCU\...\CurrentVersion\Run`（`RegSetValueExW` / `RegDeleteValueW`） |
| 单实例 | 进程启动创建 Mutex |

**新工具借鉴：** 全局快捷键注册、常驻后台架构、注册表开机自启、配置文件驱动、单实例互斥。

**新工具放弃：** 托盘图标与右键菜单（要求"无界面"）、息屏模式（要求"真黑屏"）、系统锁屏模式（改为独立密码黑屏锁）。

### 2.2 black_screen / BlackLock（黑屏锁）

**技术栈**：纯 C（C17）、纯 Win32、无任何第三方依赖、CMake；单文件 `lock.c`；链接 `user32 / gdi32 / bcrypt / shell32`。

**已实现主要功能：**

| 功能 | 实现方式 |
|------|----------|
| 全屏黑色遮挡窗口 | 覆盖整个虚拟桌面的 `WS_POPUP` + `WS_EX_TOPMOST` + `WS_EX_TOOLWINDOW` 黑窗，背景 `BLACK_BRUSH` |
| 底层全局键盘钩子 | `WH_KEYBOARD_LL` 吞掉所有按键，拦截 `Win / Alt+Tab / Alt+Esc / Alt+F4 / Ctrl+Esc` |
| 密码输入缓冲 | 钩子里把可打印字符攒进 UTF-16 缓冲，回车触发校验，退格删字，Esc 清空 |
| 密码校验 | 缓冲转 UTF-8 → `BCrypt` 计算 SHA-256 → 与目标哈希 `memcmp` 比对 |
| 密码来源 | 优先读 exe 同目录 `lock.cfg` 首行明文（即改即生效）；否则用编译期固化的 `PASSWORD_HASH_HEX` |
| 阻止睡眠/息屏 | `SetThreadExecutionState(ES_CONTINUOUS \| ES_SYSTEM_REQUIRED \| ES_DISPLAY_REQUIRED)` |
| 软阻止关机/注销 | `WM_QUERYENDSESSION` 返回 FALSE + `ShutdownBlockReasonCreate` |
| 隐藏鼠标指针 | `ShowCursor(FALSE)` + `WM_SETCURSOR` 里 `SetCursor(NULL)` |
| 保持置顶 | 1 秒定时器反复 `SetWindowPos(HWND_TOPMOST)` |
| 多屏/DPI 适配 | `SetProcessDpiAwarenessContext(PER_MONITOR_AWARE_V2)`；`WM_DISPLAYCHANGE` 时重新覆盖虚拟桌面 |
| 单实例 | `CreateMutexW` |
| 辅助工具 | `lock.exe genhash <密码>` 弹窗显示 SHA-256 |
| 安全逃生 | `Ctrl+Alt+Del`（内核级序列，用户态拦不住）→ 任务管理器结束进程 |

**新工具借鉴：** 黑屏遮挡窗口、底层键盘钩子吞键与密码缓冲、SHA-256 密码校验、防睡眠、软阻止关机、隐藏光标/保持置顶、多屏 DPI 适配、单实例、genhash 辅助。

**新工具改造：** black_screen 是"双击即锁"的一次性程序；新工具改为**常驻监听快捷键**，锁定只是其中一个状态。

### 2.3 融合结论

新工具 = **resting-screen 的常驻后台/快捷键/自启骨架 + 极简托盘（仅退出等少量项）** + **black_screen 的黑屏锁定/密码解锁内核** − **设置窗口** − **息屏与系统锁屏**。

## 3. 产品目标与范围

### 3.1 一句话定义

> 一个无界面、配置文件驱动、常驻后台的 Windows 单文件小工具：按 `Alt + L` 立即用真黑屏锁住屏幕（而非息屏），期间电脑不睡眠、动鼠标不解锁，输入密码回车解锁。

### 3.2 目标

- **G1**：极简——无主窗口、无设置界面；仅保留一个**系统托盘图标**作为可见入口（主要用于退出程序）。锁定时为全屏黑屏。
- **G2**：单文件二进制，体积尽可能小（目标 < 100 KB，力争 < 60 KB）。
- **G3**：仅 Windows 10/11 x64，不考虑跨平台。
- **G4**：C 语言优先（C17），无第三方运行时依赖。
- **G5**：所有行为通过带注释的配置文件调整（快捷键、密码、自启）。

### 3.3 范围外（Out of Scope）

- 不支持 macOS / Linux。
- 不做设置窗口、配置 GUI（配置仍走文件）。保留**极简托盘图标**仅用于退出/手动锁定等少量操作。
- 不做息屏模式、系统 `LockWorkStation` 模式。
- 不做多用户/域账户集成、企业策略下发。
- 不承诺对抗物理攻击、内核级绕过（`Ctrl+Alt+Del` 保留为逃生口，见 [§10](#10-安全性设计)）。

## 5. 系统架构设计

### 5.1 总体架构

单进程、双状态机。进程启动后先做初始化（读配置、同步自启、注册快捷键、建隐藏主窗口），随后进入**空闲常驻态**；收到快捷键后切入**锁定态**，解锁后回到空闲态。

> 实现说明：隐藏主窗口最终采用**普通隐藏工具窗口**（`WS_POPUP` + `WS_EX_TOOLWINDOW`，永不 `ShowWindow`），而非 `HWND_MESSAGE` message-only 窗口——因为 `RegisterHotKey` 的 `WM_HOTKEY` 向 message-only 窗口的投递不可靠。工具窗口同样不进任务栏 / Alt+Tab，且能可靠收到热键与托盘回调。

```
┌─────────────────────────────────────────────────────────────┐
│                        BlackLock.exe (单进程)                 │
│                                                               │
│  启动 → 单实例检查 → 加载配置 → 同步开机自启 → 注册 Alt+L      │
│         → 创建隐藏窗口 + 添加托盘图标 → 进入消息循环            │
│                                                               │
│   ┌──────────────┐   WM_HOTKEY    ┌────────────────────────┐  │
│   │  空闲常驻态   │  (未暂停时)     │       锁定态            │  │
│   │  IDLE        │ ─────────────▶ │  LOCKED                 │  │
│   │ 托盘图标常驻  │                │                        │  │
│   │ 隐藏主窗口    │  密码正确解锁   │ 全屏黑窗 + 键盘钩子      │  │
│   │ 监听快捷键    │ ◀───────────── │ 防睡眠 + 阻关机 + 隐光标 │  │
│   └──────┬───────┘                └────────────────────────┘  │
│          │ 托盘右键：开机自启 / 暂停 / 配置 / 退出               │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 状态机

| 状态 | 进入动作 | 持有资源 | 退出动作 |
|------|----------|----------|----------|
| INIT | 单实例互斥、DPI 感知、加载配置、同步自启、注册热键、建隐藏窗口、添加托盘图标 | Mutex、HotKey、HWND、托盘 `NOTIFYICONDATA` | — |
| IDLE | 仅托盘图标可见，等待 `WM_HOTKEY` / 托盘菜单命令；`g_paused` 标志控制是否响应热键 | 同上 | 未暂停时收到热键 → 转 LOCKED；托盘"退出" → 清理并结束进程 |
| LOCKED | 创建/显示全屏黑窗、装 `WH_KEYBOARD_LL`、`SetThreadExecutionState`、`ShutdownBlockReasonCreate`、隐藏光标、启动置顶定时器 | 黑窗 HWND、键盘钩子、执行状态、定时器 | 密码正确 → 卸钩子/隐藏黑窗/恢复执行状态/显示光标 → 转 IDLE |

> 设计选择：黑窗可以**每次锁定时创建、解锁时销毁**，也可以**常驻隐藏、锁定时显示**。推荐后者（常驻隐藏、`ShowWindow` 切换），避免重复注册窗口类、减少解锁后残留风险。见 [§14](#14-待决策项) D3。

### 5.3 模块划分

| 模块 | 文件（建议） | 职责 |
|------|--------------|------|
| 入口与生命周期 | `main.c` | `WinMain`、单实例、初始化编排、消息循环 |
| 配置 | `config.c/.h` | 读取/生成 `config.ini`，解析快捷键串、密码、自启开关 |
| 快捷键 | `hotkey.c/.h` | 解析 `Alt+L` 之类的组合键字符串，`RegisterHotKey` |
| 锁定窗口 | `lockwin.c/.h` | 黑窗窗口类、覆盖虚拟桌面、置顶定时器、`WM_DISPLAYCHANGE` 重铺 |
| 键盘钩子与密码 | `keyhook.c/.h` | `WH_KEYBOARD_LL`、吞键、密码缓冲、SHA-256 校验 |
| 自启 | `autostart.c/.h` | 注册表 Run 键读写 |
| 托盘 | `tray.c/.h` | `Shell_NotifyIcon` 图标、右键弹出菜单、菜单命令分发 |
| 工具 | `util.c/.h` | 路径获取、UTF-8/16 转换、SecureZero 等 |

> 注：为追求极小体积与"单文件源码"风格，也可全部内联在单个 `main.c` 中（black_screen 即如此）。模块化仅用于可维护性，二者产物体积无本质差异。见 [§14](#14-待决策项) D4。

## 6. 详细设计

### 6.1 启动与初始化流程

```
WinMain:
  1. 解析命令行：若为 --genhash <pw> → 计算并弹窗显示 SHA-256，退出
                 若为 --install/--uninstall → 写/删自启，退出（可选）
  2. CreateMutexW("Local\\BlackLock_singleton")；已存在 → 退出
  3. SetProcessDpiAwarenessContext(PER_MONITOR_AWARE_V2)
  4. 定位并加载配置：exe 同目录 config.ini 优先，缺失则 %APPDATA%\BlackLock\config.ini；都无则生成默认（见 §8.1）
  5. 根据配置 autostart 同步注册表 Run 键
  6. 解析并 RegisterHotKey(热键)
  7. 创建隐藏主窗口（收 WM_HOTKEY / 托盘回调），Shell_NotifyIcon(NIM_ADD) 添加托盘图标
  8. 注册黑窗窗口类，创建（常驻隐藏的）黑窗
  9. 进入消息循环
```

### 6.2 全局快捷键（FR-2）

- 采用 `RegisterHotKey(hwnd, id, fsModifiers, vk)`（与 resting-screen 一致），比底层键盘钩子更省资源、更规范。
- 修饰键映射：`Alt→MOD_ALT`、`Ctrl→MOD_CONTROL`、`Shift→MOD_SHIFT`、`Win→MOD_WIN`；建议附加 `MOD_NOREPEAT` 防长按连触。
- 配置字符串解析：把 `"Alt+L"` 解析成 `MOD_ALT` + 虚拟键 `'L'`。字母/数字直接取大写 ASCII；功能键做一张小映射表（`F1..F12`、方向键等）。
- 注册失败（热键被占用）处理：写日志（可选）或静默；不弹窗打扰。见 [§14](#14-待决策项) D5。

### 6.3 锁定窗口（FR-3、FR-4、FR-13）

- 窗口样式：`WS_POPUP`，扩展样式 `WS_EX_TOPMOST | WS_EX_TOOLWINDOW`（工具窗不进 Alt+Tab 列表）。
- 背景：`GetStockObject(BLACK_BRUSH)`；无边框、无标题。
- 覆盖范围：`SM_XVIRTUALSCREEN / SM_YVIRTUALSCREEN / SM_CXVIRTUALSCREEN / SM_CYVIRTUALSCREEN` 得到整个虚拟桌面并铺满。
- 置顶保持：1 秒定时器反复 `SetWindowPos(HWND_TOPMOST, SWP_NOMOVE|SWP_NOSIZE|SWP_NOACTIVATE)`。
- **让位任务管理器（保证逃生口可用）**：定时器每拍先检查前台窗口是否属于任务管理器——判定优先用**进程映像名 `Taskmgr.exe`**（`GetWindowThreadProcessId` → `OpenProcess(PROCESS_QUERY_LIMITED_INFORMATION)` → `QueryFullProcessImageNameW`，版本无关，兼容 Win11 重写后的新版任务管理器），旧版类名 `TaskManagerWindow` 作快速兜底。是任务管理器时 `SetWindowPos(hwnd, HWND_BOTTOM, ...)` 让黑窗沉到最底并放弃置顶，使其完全露出、可用鼠标"结束进程"解锁；否则维持 `HWND_TOPMOST`。这样只对任务管理器让位（不构成通用绕过），`Ctrl+Alt+Del → 取消` 返回后下一拍会重新盖上。
  - **注意（Win11 陷阱）**：早期只按类名 `TaskManagerWindow` 判定，在 Win11 新版任务管理器上失效（类名已变），导致仍被盖住——故改用进程名判定。
  - **UIPI 限制**：任务管理器以高完整性运行，非提权进程安装的低级键盘钩子在其前台时会被系统旁路，此时锁屏无法拦截其键盘输入（含 Alt+Tab）。这属 Windows 安全设计，不提权则无法改变；但任务管理器一旦露出，用鼠标结束进程即可，无需键盘。
- 热插拔/分辨率变化：`WM_DISPLAYCHANGE` 里重新读虚拟桌面尺寸并重铺。
- 光标隐藏：`WM_SETCURSOR` 中 `SetCursor(NULL)`，并 `ShowCursor(FALSE)` 直到计数 < 0。
- 鼠标"动一下不解锁"：黑窗不响应任何鼠标消息去触发解锁；解锁**唯一途径**是键盘输入正确密码 + 回车。

### 6.4 键盘钩子与密码输入（FR-6、FR-11）

沿用 black_screen 的 `WH_KEYBOARD_LL` 设计：

- 仅锁定态安装钩子，解锁后立即 `UnhookWindowsHookEx`。
- 钩子回调对每个 keydown：
  - 拦截并吞掉 `Win`、`Alt+Tab`、`Alt+Esc`、`Alt+F4`、`Ctrl+Esc`。
  - `Enter` → `PostMessage(WM_APP_ENTER)` 触发校验。
  - `Backspace` → 删除缓冲末字符。
  - `Esc` → 清空缓冲。
  - 其它键 → `ToUnicode` 翻译为字符，仅收可打印字符（≥ 0x20）存入 UTF-16 缓冲。
  - 一律返回 1（吞键），不下传给底层应用（keyup 也吞）。
- `Ctrl+Alt+Del` 属内核级安全序列，用户态无法拦截——保留为逃生口。
- **修饰键"卡住"问题及修复**：因为用 `Alt+L` 组合键触发锁定，进入锁定时 Alt 正被按住，其 keyup 会被钩子吞掉，导致解锁后系统仍认为 Alt 处于按下状态（菜单乱跳、快捷键失灵）。修复：在**进入锁定前（钩子未装）**和**退出锁定后（钩子已卸）**各用 `SendInput` 补发一次 Alt/Ctrl/Shift/Win（左右）的 keyup，清除残留修饰键状态。补发必须在钩子未激活时进行，否则合成事件会被自身钩子再次吞掉。（black_screen 无此问题，因其双击启动、锁定时无修饰键被按住。）

### 6.5 密码校验（FR-6、FR-7）

**采用明文模式**（已决策 D7）：配置文件里 `password = xxxx`，改配置即改密码，无需重编译。

- 启动/加载配置时，对明文密码算一次 SHA-256 存入内存作为目标哈希（不在内存里长期保留明文）。
- 校验流程：输入缓冲（UTF-16）→ `WideCharToMultiByte(CP_UTF8)` → `BCrypt` SHA-256 → 与目标 32 字节 `memcmp`。用完 `SecureZeroMemory` 清缓冲。
- 正确 → 解锁回 IDLE；错误 → 清空缓冲、界面无反应。

> 明文存于配置文件的安全权衡见 [§10](#10-安全性设计)。若未来需要"文件里不可见明文"，可再加 `password_mode=hash` + `--genhash`，本设计已为其预留位置但首期不实现。

### 6.6 防睡眠与阻止关机（FR-5、FR-12）

- 进入锁定态：`SetThreadExecutionState(ES_CONTINUOUS | ES_SYSTEM_REQUIRED | ES_DISPLAY_REQUIRED)`——系统与显示器都不休眠，后台任务照跑。
- 退出锁定态：`SetThreadExecutionState(ES_CONTINUOUS)` 恢复默认电源策略。
- 软阻止关机：黑窗 `WM_QUERYENDSESSION` 返回 FALSE，配 `ShutdownBlockReasonCreate`；强制关机拦不住（可接受）。

### 6.7 开机自启（FR-8）

- 位置：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`，值名 `BlackLock`，数据为 exe 绝对路径（带引号）。
- 同步策略：每次启动读配置 `autostart`——为 `true` 则确保 Run 键存在且路径正确；为 `false` 则删除 Run 键。这样"改配置文件"即可控制自启，符合无界面理念。
- 仅用 `HKCU`（无需管理员权限）。

### 6.8 单实例（FR-10）

- `CreateMutexW(NULL, TRUE, L"Local\\BlackLock_singleton")`；`GetLastError()==ERROR_ALREADY_EXISTS` 直接退出。
- 退出时 `ReleaseMutex` + `CloseHandle`。

### 6.9 系统托盘与退出（FR-16 ~ FR-20）

程序在 IDLE 态唯一的可见入口。用于解决"无主窗口时如何退出常驻进程"的问题（原方案本考虑 `--quit` 命令行，已改为托盘）。

- **图标**：`Shell_NotifyIcon(NIM_ADD, &nid)`，`uCallbackMessage = WM_APP_TRAY`，图标复用 `app.ico`（`app.rc` 内嵌，不额外增加文件）。`szTip` 显示 "BlackLock"。
- **右键菜单**：收到托盘回调且 `lParam == WM_RBUTTONUP`（或 `WM_CONTEXTMENU`）时，`SetForegroundWindow` 后 `TrackPopupMenu` 弹出菜单。**固定四项**：

  | 菜单项 | 状态 | 动作 | 对应 FR |
  |--------|------|------|---------|
  | 开机自启 | 勾选（`MF_CHECKED` 反映当前 `autostart`） | 切换 `autostart` → 写回配置文件 → 同步注册表 Run 键 | FR-17 |
  | 暂停 | 勾选（暂停中打勾） | 切换暂停：暂停时 `UnregisterHotKey`（不再响应 `Alt+L`）；恢复时重新 `RegisterHotKey` | FR-18 |
  | 配置 | — | `ShellExecuteW(L"open", <当前生效配置路径>, ...)` 用系统默认程序（记事本等）打开正在使用的那个 config.ini | FR-19 |
  | —— 分隔线 —— | | | |
  | 退出 | — | 见下方"退出清理顺序" | FR-20 |

- **"配置"打开哪个文件**：打开 [§8.1](#81-位置与编码双位置查找已决策-d6) 加载时**实际命中并保存的那个路径**（exe 同目录或 `%APPDATA%\BlackLock`），全程用变量记住该路径，保证"打开的就是正在生效的"。**不弹任何自建设置界面**——纯文件编辑。
- **"配置"编辑后的生效（已实现热重载）**：程序用 `FindFirstChangeNotificationW` 监视配置文件所在目录，配合主循环的 `MsgWaitForMultipleObjects` 等待；文件保存后自动重载并**即时应用**密码 / 快捷键 / 开机自启，无需重启（D10 已实现）。为避免编辑器"删除重建"式保存的瞬间读到半成品或误生成默认配置，重载前会先确认文件仍存在，否则跳过等待下一次通知。
- **"暂停"状态**：用一个全局标志 `g_paused` 表示；仅影响是否响应快捷键，不影响已进入的锁定态。暂停为运行期临时状态，**不写入配置文件**（重启后默认恢复监听）。
- **退出清理顺序**：`Shell_NotifyIcon(NIM_DELETE)` 删图标 → `UnregisterHotKey` → 卸载键盘钩子（若在锁定态）→ 恢复 `SetThreadExecutionState(ES_CONTINUOUS)` → 显示光标 → 销毁窗口 → 释放互斥 → `PostQuitMessage`，避免残留。
- **左键**（可选）：双击托盘不绑定或绑定"暂停/恢复"，避免误触发锁定。见 D9。
- **体积影响**：`Shell_NotifyIcon` + `TrackPopupMenu` + `ShellExecuteW` 均在已链接的 `shell32/user32` 内，新增代码约 1–2 KB，图标复用，预计总增量 < 4 KB，仍满足 NFR-1。

## 7. 技术选型说明

本节回答需求中的几个技术问题。

### 7.1 语言标准：C17 还是更新？

- **推荐：C17（ISO/IEC 9899:2018，即 `/std:c17`）**。它是 MSVC 完整、稳定支持的最新 C 标准，也是 black_screen 现用标准。
- **C23（ISO/IEC 9899:2024）** 是目前最新的 C 标准，但 **MSVC 对 C23 的支持仍不完整**（仅 `/std:clatest` 下部分特性）。为保证用 MSVC/clang-cl 都能稳定编译出最小体积产物，**不建议**在本项目上依赖 C23 特性。
- 结论：`CMAKE_C_STANDARD 17` + `CMAKE_C_EXTENSIONS OFF`。若未来只用 Clang，可评估 C23。

### 7.2 是否使用 WTL？

- **不使用。** WTL 是基于 ATL 的 **C++** 模板库，与"C 语言优先"冲突，且会引入 C++ 运行时、增大体积。
- 本项目 UI 需求极小（仅一块黑窗 + 可能一个 message-only 窗口），**纯 Win32 C** 即可，且体积最小、依赖最少。
- 结论：不引入 WTL、不引入 ATL、不引入任何 GUI 框架。

### 7.3 第三方依赖清单

**运行时第三方依赖：无。** 仅链接 Windows 系统库：

| 库 | 用途 |
|----|------|
| `user32.lib` | 窗口、消息、热键、钩子、光标 |
| `gdi32.lib` | 黑色画刷 |
| `bcrypt.lib` | SHA-256 |
| `shell32.lib` | `CommandLineToArgvW` |
| `advapi32.lib` | 注册表（开机自启） |
| `kernel32.lib`（隐式） | 互斥、文件、`GetPrivateProfileString`、执行状态 |

**FetchContent 的定位**：需求要求"用 CMake FetchContent 安装第三方依赖"。但本设计**无需任何第三方库**即可满足全部功能，这也是体积最小的方案。FetchContent 因此**默认不启用**；仅在以下可选情况保留其价值：

- 若希望用成熟的 INI 解析库（如 `benhoyt/inih`，约几百行）替代自写解析或 `GetPrivateProfileString`，可用 FetchContent 拉取。
- 若未来引入单元测试框架（如 `Unity`）。

CMake 会预留 FetchContent 接入点（注释形式），但默认路径零第三方依赖。见 [§14](#14-待决策项) D1。

### 7.4 配置解析方式

三选一（推荐 A）：

- **A（推荐）**：`GetPrivateProfileStringW / GetPrivateProfileIntW`（kernel32 自带，零代码零依赖，天然支持 `;` 行注释、`[段]`、`key=value`）。缺点：有缓存、需绝对路径、仅 `;` 行首注释。
- **B**：自写约 50–80 行 INI 解析器，完全可控，支持 `#` 与 `;` 注释、行尾注释、UTF-8 BOM 跳过。
- **C**：FetchContent 拉 `inih`。功能最全但引入外部代码。

推荐 A 为主，若需 `#` 注释或行尾注释体验再退到 B。

### 7.5 CRT 与单文件

- 目标是**单个 exe**。Windows 10/11 自带 UCRT，故动态链接 UCRT（`/MD`）不产生额外分发文件，exe 仍是"单文件"。
- 若追求绝对零系统 CRT 依赖，可 `/MT` 静态链接（体积略增）或进一步用 `/NODEFAULTLIB` + 自定义入口彻底去 CRT（改用纯 Win32 文件 API、避免 `stdio`）。作为**可选的极限瘦身**手段，非首期目标。见 [§14](#14-待决策项) D2。

### 7.6 体积优化开关（沿用 black_screen 并加强）

- MSVC：`/O1 /Os /Gy /GS-`，链接 `/OPT:REF /OPT:ICF /INCREMENTAL:NO /DEBUG:NONE`。
- MinGW/Clang：`-Os -ffunction-sections -fdata-sections -fno-asynchronous-unwind-tables -fno-unwind-tables`，链接 `-s -Wl,--gc-sections`。
- 关闭异常/RTTI（C 无关，但避免误链 C++ 运行时）。

## 8. 配置文件设计

### 8.1 位置与编码（双位置查找，已决策 D6）

配置文件按以下顺序查找，**先命中者生效**：

1. **exe 同目录** `config.ini`（便携优先，绿色软件；与 black_screen 的 `lock.cfg` 同思路）。
2. 回退：`%APPDATA%\BlackLock\config.ini`（当同目录不存在时）。

- 编码：UTF-8（读时跳过 BOM）。
- **编译期生成**：构建时在输出目录 `dist/` 生成一份默认 `config.ini`（便携发布用；已存在则不覆盖，避免清掉用户编辑）。由 CMake `POST_BUILD` 从 `src/config.default.ini` 拷贝。
- **运行期自动生成规则**（FR-15）：两处都不存在时，**只在 `%APPDATA%\BlackLock\` 生成**默认配置，**不写入 exe 同目录**——保持 exe 目录洁净。便携性由"exe 同目录 config.ini 优先级最高"保证：想便携就把 `config.ini` 放在 exe 旁（如 `dist/` 里那份）。
- **默认模板单一事实来源**：`src/config.default.ini`。它同时被内嵌为 exe 的 `RCDATA` 资源（运行时写 `%APPDATA%` 用）并被 CMake 拷到 `dist/`，两处内容永不漂移。
- 加载伪代码：

```
path = exeDir + "config.ini"          # 1) 便携优先
if not exists(path):
    path = appData + "\\BlackLock\\config.ini"   # 2) 回退
if not exists(path):                  # 3) 都没有 -> 只在 APPDATA 生成
    ensureDir(appData + "\\BlackLock")
    write appData + "\\BlackLock\\config.ini" from 内嵌资源 BL_DEFAULT_CONFIG
    path = appData + "\\BlackLock\\config.ini"
load(path)
```

### 8.2 配置项

| 段 | 键 | 类型 | 默认 | 说明 |
|----|----|------|------|------|
| `[general]` | `hotkey` | 字符串 | `Alt+L` | 触发锁定的全局快捷键 |
| `[general]` | `autostart` | 布尔 | `false` | 是否开机自启（同步到注册表 Run） |
| `[security]` | `password` | 字符串 | `ciqtek` | 明文密码（改完保存即时热重载生效） |

### 8.3 默认配置文件样例（带注释）

```ini
; ============================================================
;  BlackLock 配置文件  (UTF-8)
;  改完保存即生效（下次锁定 / 下次启动读取）
; ============================================================

[general]
; 触发黑屏锁的全局快捷键。
; 修饰键可用 Alt / Ctrl / Shift / Win，用 + 连接，例如：
;   Alt+L      Ctrl+Alt+L      Win+L(不建议，与系统冲突)
hotkey = Alt+L

; 是否开机自启动。true = 写入注册表启动项；false = 移除。
autostart = false

[security]
; 解锁密码（明文）。改完保存即时生效（热重载）。默认 ciqtek。
; 注意：明文存储，能读到本文件的人即可看到密码，仅作临时遮挡用途。
password = ciqtek
```

> 说明：`GetPrivateProfileString` 只识别 `;` 行注释。若最终选用自写解析器（[§7.4](#74-配置解析方式) B），可同时支持 `#` 注释与行尾注释。
>
> 配置文件按 [§8.1](#81-位置与编码双位置查找已决策-d6) 的顺序在 exe 同目录 / `%APPDATA%\BlackLock` 中查找，同目录优先。

## 10. 安全性设计

> **定位申明**：BlackLock 是"防君子不防小人"的**便捷临时遮挡锁**，用于短暂离开时挡住屏幕、防止旁人误触/窥屏，**不是**对抗专业攻击者的安全产品。

| 主题 | 设计与风险 |
|------|-----------|
| `Ctrl+Alt+Del` 逃生口 | 内核级安全序列，用户态钩子无法拦截。忘记密码或程序卡死时，经任务管理器结束进程即可解锁。这是**有意保留**的安全阀，也意味着能物理接触键盘者可绕过锁定。**实现注意**：黑窗每秒重置顶会盖住任务管理器让逃生口失效，故检测到任务管理器（按进程名 `Taskmgr.exe`）在前台时主动沉到最底 `HWND_BOTTOM`（见 [§6.3](#63-锁定窗口fr-3fr-4fr-13)）。任务管理器高完整性运行，此时低级键盘钩子被系统旁路，用**鼠标**结束进程即可。 |
| 明文密码 | 密码明文存在 `config.ini`（同目录或 `%APPDATA%`），任何能读该文件者可见。这是本项目**有意选择**的便捷方案（D7），仅作临时遮挡用途。若未来需要，可扩展 SHA-256 哈希模式，但注意无加盐哈希仍可被暴力破解，仅"不肉眼可见"级别。 |
| 内存中的密码 | 输入缓冲与转换缓冲用完 `SecureZeroMemory` 清零。 |
| 进程被杀 | 无自恢复守护；被任务管理器结束即解除锁定（与逃生口一致）。 |
| 关机/注销 | 仅软阻止（`WM_QUERYENDSESSION`），强制关机/断电不可拦。 |
| 权限 | 全程仅需普通用户权限（`HKCU`、同目录配置）。不申请管理员，避免 UAC 与越权风险。 |
| 数据外发 | 无任何网络行为，纯本地。 |

**文档诚实声明**：由于 `Ctrl+Alt+Del` 无法拦截，本工具**无法阻止**有物理键盘访问权的人结束进程解锁。若需真正的安全锁定，应使用系统自带 `Win+L` / 账户密码 / BitLocker。

## 12. 风险与已知限制

| 风险/限制 | 说明 | 缓解 |
|-----------|------|------|
| `Ctrl+Alt+Del` 无法拦截 | 用户态限制 | 文档明示；定位为便捷锁而非安全锁 |
| 明文密码可见 | 便捷性代价（已选明文） | 文档警示；定位临时遮挡；可后期扩展 hash 模式 |
| 热键被其它软件占用 | `RegisterHotKey` 失败 | 允许配置改键；失败静默/可选日志 |
| Program Files 目录只读 | 同目录 config 写入失败 | 自动回退 `%APPDATA%\BlackLock`（D6 已解决） |
| MSVC C23 支持不全 | 语言标准 | 锁定 C17 |
| 极限去 CRT 复杂 | 体积极致优化 | 作为可选项，非首期 |
| 全屏钩子被安全软件误报 | `WH_KEYBOARD_LL` 常被杀软关注 | 说明用途；必要时数字签名 |
